# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of our systems seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Disclosure Policy

*   Please do not report security vulnerabilities through public GitHub issues.
*   Please report them to the security team via email at **security@linkdao.io**.
*   We will acknowledge your report within 24 hours.
*   We will provide an estimated timeframe for addressing the vulnerability.
*   We ask that you do not disclose the issue publicly until we have had a reasonable time to address it.

### What is in scope?

*   LinkDAO Backend API
*   LinkDAO Smart Contracts
*   LinkDAO Frontend logic

### What is out of scope?

*   DDoS attacks
*   Social engineering
*   Third-party services or dependencies (unless a specific implementation issue)

## Incident Response

We have a detailed Incident Response Plan. In case of a confirmed breach, we will follow our internal procedures to contain, investigate, and resolve the issue, including notifying affected users as required by law (e.g., GDPR).

## Security Best Practices

*   We use Semantic Versioning.
*   We conduct regular security audits.
*   We enforce 2FA and strong authentication for administrative access.
